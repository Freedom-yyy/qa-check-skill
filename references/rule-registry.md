# QA 规则注册表

本文件定义 `qa-check` 可写入 JSON `ruleIds` 的稳定规则。执行者只选择有触发证据的规则，并在 `ruleExecutionSummary` 中记录执行状态和结果。没有对应注册项时使用 `general.diff-review`，不得临时编造规则 ID。

规则状态：`Executed`、`NotApplicable`、`BlockedByMissingMaterials`、`Skipped`。
规则结果：`IssueFound`、`NoIssueFound`、`CannotDetermine`。

## general.diff-review

- 领域：通用代码审查
- 触发条件：所有非空 diff
- 检查内容：变更逻辑、调用链、异常处理、兼容性和直接回归风险
- 证据要求：确定问题必须能由代码路径定位；无法定性时降为疑似问题或测试场景

## frontend

- 领域：前端
- 触发条件：前端页面、组件、状态、路由或交互代码发生变化
- 检查内容：输入约束、状态流转、展示值变换、生命周期、异步交互和用户可操作性
- 证据要求：区分代码可证实问题、需运行态确认的风险和建议测试场景

## backend

- 领域：后端
- 触发条件：服务、控制器、业务逻辑、任务或数据访问代码发生变化
- 检查内容：参数校验、异常处理、幂等性、状态流转、事务和失败恢复
- 证据要求：说明请求路径、数据状态和可观察影响；缺少依赖资料时标记无法判断

## api-contract

- 领域：接口契约
- 触发条件：接口路径、请求参数、响应字段、错误码或序列化逻辑变化
- 检查内容：调用方兼容性、字段语义、空值/异常响应和版本影响
- 证据要求：有调用方或接口资料时做一致性判断，否则记录资料缺口

## database

- 领域：数据库
- 触发条件：迁移、模型、查询、索引或数据写入逻辑变化
- 检查内容：老数据、空值、唯一性、迁移兼容、回滚和查询结果影响
- 证据要求：区分代码证据与需要真实数据/数据库环境验证的风险

## auth-permission

- 领域：鉴权与权限
- 触发条件：登录态、角色、权限、租户或资源访问判断变化
- 检查内容：越权、漏权、老账号兼容和不同身份的路径差异
- 证据要求：必须说明账号角色或权限上下文；缺少账号时不得宣称已验证

## stateful-flow

- 领域：状态流转
- 触发条件：恢复、跳转、缓存、进度、初始化、状态字段或短路入口变化
- 检查内容：被跳过流程的副作用是否恢复，以及所有真实下游消费方
- 证据要求：没有真实消费方或业务预期不明确时，不直接报为确定问题

## async-interaction

- 领域：异步交互
- 触发条件：保存、提交、加载、重试、消息或其他耗时操作变化
- 检查内容：loading、禁用态、重复触发、失败提示、回滚和重试能力
- 证据要求：弱网和时序表现无法由代码确定时，落为疑似问题或测试场景

## visual-print

- 领域：视觉与打印
- 触发条件：样式、字体、分页、打印、导出或 PDF 逻辑变化
- 检查内容：视觉验证矩阵、分页、截断、溢出、打印和导出结果
- 证据要求：未取得测试入口、截图或 PDF 时，不将视觉差异写成确定问题

## configuration

- 领域：配置与灰度
- 触发条件：环境变量、Feature Flag、构建配置或运行时开关变化
- 检查内容：默认值、环境差异、灰度状态和开关关闭时的兼容性
- 证据要求：缺少环境配置时标记 `BlockedByMissingMaterials` 或 `CannotDetermine`

## project-hotspot

- 领域：历史高风险区域
- 触发条件：本项目规则 reference 或用户提供的历史问题证据明确命中；命中后才读取对应 reference
- 检查内容：只执行已脱敏、已注册且与当前变更相关的历史风险模式
- 证据要求：必须记录 reference、触发依据、实际检查动作和结论边界；不得把具体项目案例泛化为无条件规则，也不得仅以“已加载”代替执行记录
